# Cloud Vs On-Prem

Cloud and on-prem are two broad ways organisations run technology systems. The important question is not which one sounds more modern. The important question is what responsibilities, costs, risks, and constraints come with each option.

## The Difference

<div class="de-article-compare">
  <article>
    <h3>On-prem</h3>
    <p>Systems run on infrastructure owned or directly managed by the organisation. This can offer control, but requires maintenance, capacity planning, physical responsibility, and upgrade work.</p>
  </article>
  <article>
    <h3>Cloud</h3>
    <p>Systems run on infrastructure provided over the internet. This can offer flexibility and scale, but still needs cost control, security, governance, and good design.</p>
  </article>
</div>

## Shared Responsibility

<div class="de-data-type-grid">
  <article>
    <span>Security</span>
    <h3>Who protects what?</h3>
    <p>Cloud providers secure parts of the platform, but organisations still manage access, data, and configuration.</p>
  </article>
  <article>
    <span>Cost</span>
    <h3>Who watches usage?</h3>
    <p>Cloud can scale quickly, which is useful. It can also become expensive if nobody monitors it.</p>
  </article>
  <article>
    <span>Recovery</span>
    <h3>Who restores service?</h3>
    <p>Backups, recovery plans, and testing still matter wherever a system runs.</p>
  </article>
  <article>
    <span>Change</span>
    <h3>Who maintains it?</h3>
    <p>Updates, patches, configuration, and monitoring need clear ownership.</p>
  </article>
</div>

## Mini Challenge

For a system you use at work, ask: where does it run, who manages access, and what would happen if it became unavailable?

<section class="de-scenario-task">
