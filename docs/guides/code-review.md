# Code Review Essentials

A code review is not a personal judgement. It is a quality conversation about whether a change is clear, safe, maintainable, and fit for purpose.

## What Reviewers Check

<details class="de-check-understanding" open>
  <summary>Review checklist</summary>
  <ol>
    <li>Does the change solve the right problem?</li>
    <li>Is the code understandable to someone else?</li>
    <li>Are there obvious bugs, risks, or edge cases?</li>
    <li>Are tests or checks included where they matter?</li>
    <li>Is anything more complicated than it needs to be?</li>
  </ol>
</details>

## Better Feedback

<div class="de-article-compare">

## Receiving Review

<div class="de-callout-grid">
  <article>
    <h3>Pause first</h3>
    <p>Feedback can feel personal. Give yourself a moment before replying.</p>
  </article>
  <article>
    <h3>Ask why</h3>
    <p>If a comment is unclear, ask what risk or confusion the reviewer sees.</p>
  </article>
  <article>
    <h3>Update clearly</h3>
    <p>Respond with what changed so the reviewer can check it quickly.</p>
  </article>
</div>
