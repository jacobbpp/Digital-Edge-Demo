# Incidents, Bugs, And Problems

Technology teams need a shared language for things going wrong. Clear language helps people respond faster and avoid confusing the symptom with the cause.

## Key Terms

<div class="de-role-grid">
  <article>
    <strong>Bug</strong>
    <span>A fault in software or configuration that causes incorrect behaviour.</span>
  </article>
  <article>
    <strong>Incident</strong>
    <span>An event that disrupts service, users, security, or business operations.</span>
  </article>
  <article>
    <strong>Root cause</strong>
    <span>The underlying reason something happened, not just the visible symptom.</span>
  </article>
</div>

## Symptom Vs Cause

<div class="de-article-compare">
  <article>
    <h3>Symptom</h3>
    <p>Learners cannot submit a form.</p>
  </article>
  <article>
    <h3>Possible cause</h3>
    <p>A required field was changed, but the validation message was not updated.</p>
  </article>
</div>
