# Technical Judgement

Technical judgement is the ability to choose a reasonable path when there are trade-offs. Most real decisions involve speed, cost, risk, quality, user needs, and maintainability.

## Trade-Offs Are Normal

<div class="de-data-flow">
  <span>Speed</span>
  <span>Cost</span>
  <span>Risk</span>
  <span>Quality</span>
  <span>Maintainability</span>
</div>

Good technical judgement does not mean finding a perfect answer. It means understanding the trade-off and explaining it honestly.

## Before Recommending A Solution

<div class="de-check-card">
  <h2>Ask:</h2>
  <label><input type="checkbox"> What problem are we solving?</label>
  <label><input type="checkbox"> Who is affected by this decision?</label>
  <label><input type="checkbox"> What could go wrong?</label>
  <label><input type="checkbox"> How will we know it worked?</label>
  <label><input type="checkbox"> What will this be like to maintain later?</label>
</div>
