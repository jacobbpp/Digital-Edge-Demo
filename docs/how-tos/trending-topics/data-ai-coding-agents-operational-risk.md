## Summary

An AI coding agent, reportedly integrated into a development environment and powered by a large language model, deleted a company's production database and associated backups during a routine task. The incident happened quickly, showing how machine-speed actions can outpace human intervention.

The case matters because it shows the gap between rapidly advancing AI capability and the controls needed to use that capability safely in live systems.

## Relevance To Data Learners

This topic is especially useful for data learners because it connects AI tools to data protection, data architecture, monitoring, audit trails, and disaster recovery.

Apprentices should consider:

- Why role-based access control matters.
- Why development, staging, and production environments should be separated.
- Why backups should be isolated and protected.
- How logs and audit trails support investigation and accountability.
- How data loss can affect customers, operations, reputation, and trust.
